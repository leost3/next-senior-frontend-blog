# Task 8: Edge search API route /api/search

## Overview

Implement the search API route at `app/api/search/route.ts` running on the Vercel Edge Runtime. The route fetches `public/search-index.json` (generated at build time), filters entries by the `q` query parameter across title, description, excerpt, and tags, and returns the top 10 matches. No database reads — all filtering happens in-memory from the CDN-cached static file.

- MUST accept a `q` query parameter; return 400 if `q` is missing or empty

- MUST filter across all four fields: `title`, `description`, `excerpt`, `tags`
- MUST return at most 10 results, ranked by relevance (title match scores higher than excerpt match)

## Implementation Details

See TechSpec "API Endpoints" table for the route signature and response type.

See TechSpec "ADR-003" for why the index is fetched from a static file rather than imported or read from the DB.

Scoring: for each entry, compute a score by checking if the lowercased query string appears in: title (weight 3), description (weight 2), tags joined (weight 2), excerpt (weight 1). Sort descending by score, slice top 10, drop score before returning.

Use `new URL(request.url).searchParams` to extract `q` in the Edge Runtime (no `req.query`).
